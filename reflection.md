# Reflection

1. Git is a version control tool that tracks changes to files locally. GitHub is a cloud platform for hosting Git repositories and collaborating with others online.

2. The terminal lets you interact with your computer using text commands instead of a graphical interface, giving faster and more precise control over files, programs, and tools like Git.

3. git add stages changes, marking them to be included in the next commit.

4. git commit saves the staged changes as a permanent snapshot in the project's history, along with a message describing what changed.

5. Daily Git workflow:
   - Check status: git status
   - Stage changes: git add .
   - Commit changes: git commit -m "message"
   - Pull latest changes (if collaborating): git pull
   - Push to GitHub: git push
